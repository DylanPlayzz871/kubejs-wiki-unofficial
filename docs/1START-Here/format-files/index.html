Learn to format your files

There is a LOT of things in kubejs that NEED to be in the right folders. You could have all your code correct and nothing will work. Things like

  • recipe scripts
  • actual game function changes
  • functions on player join
  • command creation
  • etc

Lets start off by learing where you can find your mod files and kubejs files

The biggest thing you need to know is how to find your game files. For me im going to show this in CurseForge but it may differ from launcher to launcher

Lets start by actually going into your modpack in CurseForge and right next to the play button you will see 3 little dots

Now click on open folder

Entire your mod file folder​

Now that we are in your modpack directory you need to find a folder called Kubejs, that is where ALL of your work will be done

Inside the Kubejs folder you will see a few different folders and a ReadMe file, the few files you will more than likely be worried about will be the server_scripts and startup_scripts. This will be where most of your code will be ran, made, or messed with

Both the Server_Script and the startup_scripts folder will usually have a file in them by default. In different version they may be names something different but in this example they are called main.js in the startup_scripts and main.js in the server_scripts but are sometimes names example.js

Creating a new .js script​

To start making your own file or script, it is very simple. You can start out by using the example one that is created for you at first launch but when you are ready to create a NEW file it is very simple.

Start out by going into one of the two folders that we spoke about before (startup_scripts/Server_scripts)

Start out by right clicking anywhere in the folder and go down to NEW

then click on new text document

Once you get a new file, rename it to something relative to what you are changing and change the .txt to .js

When clicking ENTER it will ask you if you are sure you would like to change the file type from .txt to .js, just click yes

Item name and namespaces​

When coding in kubejs or any coding related things for minecraft you are going to need to know the namespaces of items, blocks, machines, fluids, etc.

To easier find them when you are in game Press F3 + H to enable advanced tool tips in game and Now when you hover over any block, item, fluid, etc you will see a new name show up looking like minecraft:oak_log instead of just oak_log

these are the names you will be using in code to tell the game what items you are wanting to use.
